# DayNote — Impeccable Design Context

## Design Context

### Users
知识工作者：频繁参与会议、整理会议记录和文档，每天产生多条日志。使用场景：快速打开回顾昨天的会议内容，检索某次讨论的决策。核心任务是**高效浏览 + 沉浸阅读**，不是创作。

### Brand Personality
**克制 · 沉静 · 专业**

- Bear / Things 3 基调：深灰或纯白背景，大量留白，内容主导
- 只有一种 accent 色（当前 `#ff3b30` 红色，用于录音按钮和选中态）
- 无阴影、无渐变、无多余装饰
- 情感目标：打开即平静，阅读不费力，每次回顾都是享受

### Aesthetic Direction
- **视觉基调**：macOS 原生 / Bear-style minimal
- **字体**：`-apple-system, SF Pro Text`（已在用，继续保持）
- **参考**：Bear App、Things 3、Notch、Linear（仅密度参考）
- **反参考**：Notion 的彩色 banner / 卡片阴影；Obsidian 的插件堆砌感
- **主题**：以深色模式为优先呈现质量基准（截图显示用户主要用深色）

### Design Principles

1. **留白即信息**：段落间距、行高比字号更重要。正文宜用 `line-height: 1.9`，段落之间加 `margin-bottom: 10px`
2. **层级靠尺寸，不靠颜色**：h1/h2/h3/正文的字号差要明显（18 / 15 / 11 / 13），避免同一灰色系的颜色堆叠
3. **代码可操作**：代码块加复制按钮；inline code 颜色用中性（不要红色），保持克制
4. **过渡有温度**：加载中的 Spinner、空状态插图要与整体深色调协调，避免生硬闪烁
5. **单一 accent**：全局只有录音红色 `#ff375f` 作为 accent，其余交互用透明度变化，不引入第二种彩色

## UI Improvement Roadmap

### Markdown 渲染优化（优先级高）

#### 可读性
- [ ] 正文行高从 `1.85` 提升到 `1.9`，段落 margin-bottom 从 `6px` → `10px`
- [ ] h1 字号：`17px` → `18px`；h2 字号：`14px` → `15px`（与正文 13px 拉开差距）
- [ ] 正文颜色深色模式：`#c8c8c8` → `#d4d4d4`（提升对比度，减少眼疲劳）

#### 信息层次
- [ ] h2 去掉 border-bottom，改用更大的 `margin-top: 24px`（留白比横线更克制）
- [ ] h3 保持全大写 + letter-spacing，但颜色用 `#8e8e93` 而非 `#636366`（稍亮）
- [ ] 有序列表加 `counter` 样式，数字用 accent 颜色或单独字重区分

#### 代码块
- [ ] 添加**复制按钮**（右上角，hover 显示）
- [ ] Inline code 颜色改为中性：深色模式用 `#a8b5c8`（蓝灰），替换当前 `#ff6b6b` 红色
- [ ] 代码块背景在深色模式用 `#252528`（比当前 `#2c2c2e` 深一点，与正文区分更明显）

#### 空状态 & 加载
- [ ] 空状态图标从 📝 emoji 改为 SVG 线框图标（与克制风格一致）
- [ ] 加载 Spinner 替换为骨架屏（skeleton lines）或淡入动效，避免居中转圈的突兀感

### 列表区（左侧）优化

- [ ] 选中态左侧 bar 宽度从 `2px` → `3px`，更有存在感
- [ ] Tag 标签在深色模式下背景透明度从 `0.10` → `0.12`，稍微可见
- [ ] 日期列（28 / 周六）字号层次：日期数字可用 `16px`，星期用 `9px`

### 整体布局

- [ ] 分割线 `DIVIDER_WIDTH: 7px` 偏宽，改为 `5px` 更精致
- [ ] 右侧 detail panel 内容区 `padding: 16px 20px` → `padding: 20px 24px`（呼吸感）
