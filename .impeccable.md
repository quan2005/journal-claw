## Design Context

### Users

知识工作者：频繁参与会议、整理文档，每天产生多条日志。使用场景是会后 5 分钟快速回顾、周末批量浏览。核心任务是**高效浏览 + 沉浸阅读**，不是创作。他们对工具的期望是「打开即平静，阅读不费力」——像翻开一本装帧讲究的笔记本，而非面对一个管理系统。

### Brand Personality

**克制 · 沉静 · 专业**

- 情感目标：每次打开感到安心而非焦虑，阅读时忘记工具的存在
- 气质对标：Aesop 的包装克制、Dieter Rams 的功能纯粹、Bear App 的阅读沉浸
- 三个词：**Intentional · Quiet · Precise**

### Aesthetic Direction

- **参考**：Bear App（阅读沉浸）、Things 3（交互精度）、Linear（信息密度）、Aesop（克制的高级感）
- **反参考**：
  - Notion 彩色 banner / 卡片阴影 / 插件堆砌
  - AI slop 指纹：紫蓝渐变、霓虹 accent、玻璃态、bounce 缓动、装饰性 sparkline
  - 万物皆卡片、卡片套卡片
  - Hero 指标模板（超大数字 + 小标签 + 渐变强调）
- **主题优先级**：深色模式为主要质量基准；浅色模式同等打磨
- **高级感路径**：**Quieter** — 奢侈品式精密克制，每个像素都有意图但绝不张扬

### Design Principles

1. **留白即信息** — 行高、段落间距比字号更重要。紧密（8–12px）用于组内关联，宽松（32–48px）用于章节分隔。留白创造节奏，不是浪费空间
2. **层级靠字号 + 字重，不靠颜色** — h1/h2/h3/正文用字号差 + 字重差建立层级，颜色仅用于语义（accent、状态、链接）
3. **单一 accent** — 全局只有录音红色（`#ff3b30` / `#ff375f`），用于录音按钮、选中态等关键强调。其余交互用透明度变化和 tinted neutral 色阶。禁止引入第二种彩色 accent
4. **字体有意图** — 正文用系统无衬线（SF Pro），代码块用 IBM Plex Mono，不混用。等宽字体不是品牌气质，而是代码语义
5. **动效有纪律** — 只动 transform + opacity，反馈 ≤150ms，过渡 200–300ms，入场 ≤500ms。缓动用 ease-out 家族（cubic-bezier(0.16, 1, 0.3, 1)）。永远尊重 prefers-reduced-motion
6. **Neutrals 带墨水青调** — 灰色系带微妙的冷青 tint（ink cyan），介于冷暖之间，赋予中性色品质感而非死灰
7. **Anti-slop 自检** — 如果别人一眼说「这是 AI 做的」，就是失败。拒绝：紫蓝渐变、bounce 缓动、装饰性模糊、渐变文字、多余圆角发光

### Typography System


| 角色          | 字族                                                         | 说明                               |
| ----------- | ---------------------------------------------------------- | -------------------------------- |
| **Body**    | `system-ui, -apple-system, BlinkMacSystemFont, sans-serif` | macOS 上自动使用 SF Pro Text；无需额外加载字体 |
| **Display** | 同上（SF Pro 在大字号自动切换 Display 光学尺寸）                           | 标题 ≥20px 时系统自动优化                 |
| **Mono**    | `'IBM Plex Mono', ui-monospace, monospace`                 | 仅用于代码块和 inline code              |


**字号标尺**（模块化比例 1.25）：


| Token         | 尺寸               | 用途                   |
| ------------- | ---------------- | -------------------- |
| `--text-xs`   | 0.75rem (12px)   | 辅助信息、时间戳             |
| `--text-sm`   | 0.8125rem (13px) | 次要文字、meta            |
| `--text-base` | 0.875rem (14px)  | 列表正文                 |
| `--text-md`   | 1rem (16px)      | 详情面板正文、Markdown body |
| `--text-lg`   | 1.25rem (20px)   | 小标题                  |
| `--text-xl`   | 1.5rem (24px)    | 页面标题                 |
| `--text-2xl`  | 1.875rem (30px)  | 大标题（罕用）              |


**字重角色**：400 正文 / 500 强调 / 600 标题。不使用 300 或 700+。

### Color System

**单一 accent + tinted neutral 体系：**


| 角色                 | 浅色模式        | 深色模式        | 说明           |
| ------------------ | ----------- | ----------- | ------------ |
| **Accent**         | `#ff3b30`   | `#ff375f`   | 录音、选中、关键强调   |
| **Background**     | ink-cyan 浅灰 | ink-cyan 深灰 | 带微妙冷青色调      |
| **Text Primary**   | 近黑（非纯黑）     | 近白（非纯白）     | 降低极端对比的刺眼感   |
| **Text Secondary** | 中灰          | 中灰          | meta、辅助信息    |
| **Border/Divider** | 极浅灰         | 极深灰         | 用于结构分隔，几乎不可见 |


**语义色（仅用于语义，非装饰）：**

- Success: 柔和绿（确认、完成）
- Warning: 柔和琥珀（注意、处理中）
- Error: 与 accent 红一致

**标签色策略**：标签不使用鲜艳多色盘。用 accent 红的透明度变化 + tinted neutral 色阶，保持视觉统一。

### Animation Discipline

- **仅动 transform + opacity** — 禁止动 width/height/padding/margin
- **时长**：即时反馈 100–150ms，hover/菜单 200ms，布局变化 300ms，入场 400–500ms
- **缓动**：`cubic-bezier(0.16, 1, 0.3, 1)`（ease-out-quart），禁止 bounce/elastic
- **退出比进入快 25%**
- **prefers-reduced-motion**：必须尊重，降级为 opacity crossfade 或直接切换
- **hero moment**：每个流程最多一个精心编排的过渡（如录音开始/结束），其余保持克制

### Anti-Patterns Checklist

在实现 UI 时逐项自检：

- 是否使用了渐变文字或霓虹发光？→ 删除
- 是否有 bounce/elastic 缓动？→ 换成 ease-out
- 是否动了 width/height/margin？→ 改用 transform
- 是否有纯灰（无色调 tint）的大面积表面？→ 加墨水青 tint
- 是否有「卡片套卡片」？→ 扁平化层级
- 标题和副标题是否说了同一件事？→ 删除重复
- 是否所有按钮都是主按钮？→ 加入 ghost/text 层级
- 是否用颜色作为唯一区分手段？→ 加图标/文案辅助
- 是否使用了纯 #000 或 #fff？→ 替换为 tinted near-black/white
