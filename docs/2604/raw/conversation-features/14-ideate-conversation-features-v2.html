展示层
1

Markdown 富文本渲染

assistant 消息从 pre-wrap 纯文本升级为完整 Markdown 渲染 — 标题、列表、加粗、行内代码、代码块（带语法高亮）。流式输出时实时渲染，不等完成。
技术路径：前端引入轻量 MD 渲染（如 marked + highlight.js），仅对 assistant 消息生效。用户消息保持纯文本气泡。

交互层
3

斜杠快捷命令

输入 / 弹出命令面板。预置命令直接注入 prompt 模板，减少自然语言描述成本。命令可带参数。
命令列表可扩展，后续支持用户自定义。面板用键盘上下选择 + Enter 确认。

🏷️ /tags — 重新生成标签
4

上下文自动注入

打开对话时自动将当前选中日志作为上下文。输入框上方显示上下文 chip，可移除。支持拖拽文件追加更多上下文。
Rust 层：conversation_create 已有 context 参数，扩展为结构化的 context_files。前端：从 selectedEntry 读取内容传入。

体验层
5

会话持久化与历史

对话历史保存为日志的 sidecar 文件（.conversations/），下次打开同一篇日志时可恢复。对话框顶部增加会话切换器。
存储格式：JSON，包含 messages + metadata（时间、关联日志）。列表按时间倒序，最近的在上。

6

富输入框

参考底部 CommandDock 的交互模式，让对话输入框支持：拖拽文件附件、粘贴图片/文件、多行自适应高度、附件预览 chip。从「纯文本框」进化为「素材收集器」。
复用 CommandDock 的 FileCard 组件和剪贴板路由逻辑。输入框左侧加 + 按钮，点击可选择文件。
