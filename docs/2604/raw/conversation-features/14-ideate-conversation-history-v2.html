会话标题 · LLM 动态总结

第一轮对话完成后，自动用一次轻量 LLM 调用生成简短标题。
标题随对话深入可以更新，始终反映核心话题。

标题生命周期
1

新建 — 占位标题

会话刚创建时，左栏显示 新对话 占位文字，或显示 shimmer 骨架屏。用户开始输入第一条消息。

2

首轮完成 — 触发总结

第一轮 assistant 回复结束（done 事件）后，后台发起一次独立的 LLM 调用：
"用 ≤8 个中文字总结这段对话的主题"
不阻塞对话，标题异步更新。

3

标题就位 — 平滑替换

LLM 返回标题后，左栏的占位文字 fade 过渡为真实标题。同时写入持久化文件。右栏 header 同步更新。

4

话题漂移 — 可选更新

如果对话进行了 5+ 轮且话题明显偏移，可以再次触发总结更新标题。或者用户双击标题手动编辑。

⚡

轻量调用，不阻塞

标题总结是独立的 LLM 请求，只传入第一轮的 user + assistant 消息，system prompt 极简："用≤8个中文字总结主题，只输出标题"。用最小 token 完成，不影响对话主流程。

🔄

事件驱动更新

Rust 层新增 conversation-title 事件，payload 为 { session_id, title }。前端监听后更新左栏和右栏 header。标题生成失败时 fallback 为用户首条消息的前 15 字。

✏️

可手动覆盖

左栏标题支持双击进入编辑态（inline edit），用户可以手动修改。手动修改后标记 title_locked: true，后续不再自动更新。

💾

持久化结构

标题存入会话 JSON 文件：{ "id": "conv-xxx", "title": "Q3规划会议分析", "title_locked": false, "linked_entry": "2604/14-AI平台会议.md", "messages": [...] }
