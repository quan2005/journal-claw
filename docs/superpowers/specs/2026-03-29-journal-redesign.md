# Journal 重定位设计

## 定位转变

从**录音工具** → **智能日志平台**。

Journal 是个人笔记库（notebook）的输入前端：将多种素材通过 AI 整理成符合 notebook 规范的 markdown 文件，存入 `{workspace}/yyMM/` 目录。

---

## 核心概念

### 两类对象

| 对象 | 说明 |
|------|------|
| **原始素材** | 用户输入的原始内容：录音、txt、md、pdf、docx |
| **日志条目** | AI 整理后生成的 markdown 文件，存入 `{workspace}/yyMM/dd-主题.md` |

### 存储结构

```
{workspace}/
└── yyMM/               # 按年月分目录，如 2603/
    ├── 28-AI平台产品会议纪要.md
    ├── 28-下午公园散步.md
    └── ...
```

原始素材统一存放在 `{workspace}/yyMM/` 下的 `raw/` 子目录（与日志条目同月归档）。

### 日志格式

遵循 notebook 秘书稿规范：

```markdown
---
tags: [journal, meeting]
summary: "结论 + 背景 + 价值/影响"
---

# 标题

## 背景
## 关键讨论 / 核心内容
## 结论
## 行动项
```

- frontmatter 只保留 `tags` 与 `summary`
- `summary` 写 1-3 句，先结论后背景
- 正文结构依据内容类型选用对应模板（meeting / design / report / goal / plan / research / guide）

---

## AI 整理机制

### 触发时机
**实时触发**：每次添加素材后立即触发 AI 处理。

### AI 引擎
通过配置的**命令行**调用，应用本身不绑定特定 AI 服务。设置界面提供命令行配置入口。

### 整理粒度
AI 智能决定：
- 新素材可能**创建新日志条目**
- 也可能**追加/修改当天已有条目**（如同一天的多段录音合并为一篇会议纪要）

### 用户编辑保留
用户可编辑 AI 生成的日志内容。下次 AI 更新时**智能保留**用户手动修改的部分。

## UI 设计

### 主界面布局

```
┌─────────────────────────┐
│ ● ● ●   Journal         │  ← TitleBar (36px)
├─────────────────────────┤
│ [整理中] 🎙录音 📄会议   │  ← 处理中条（仅有素材处理时显示）
├─────────────────────────┤
│ 2026年3月 ─────────────  │  ← 月份分隔线
│                         │
│ 28  AI平台产品会议纪要  会议│  ← 首条目显示日期
│ 周六 探索可继续，需同步...│
│     10:15  🎙×2 📄×1   │
│ ─────────────────────── │
│     下午公园散步 & 感想  │  ← 同天后续条目无日期
│     樱花开了，回忆童年...│
│     14:32  🎙×1        │
│ ─────────────────────── │
│ 26  《原则》读书笔记  阅读│
│ 周四 Dalio 决策五步...   │
│     09:30  📄×2        │
│                         │  ← 月末最后一条，无分隔线
│ 2026年2月 ─────────────  │
│ ...                     │
├─────────────────────────┤
│         ●               │  ← FAB 录音按钮
└─────────────────────────┘
```

### 列表条目结构

每条日志条目显示：
- **日期列（28px）**：首条显示 `日数字(15px bold)` + `周几(10px 灰)`，同天后续条目留空
- **标题**：加粗，内联标签 pill（仅有标签时显示，颜色对应类型）
- **摘要**：1行，`summary` 字段截断
- **元信息**：时间 + 来源素材计数（`🎙×2 📄×1`）

### 月份分隔线

```
2026年3月 ─────────────────
```

- 标签 `12px font-weight:600 color:#8e8e93`
- 右侧延伸线 `height:1px background:#e5e5ea`
- 月份内最后一条**不加**下方分隔线，由下一个月的标题自然隔开

### 标签 Pill

内联在标题文字后，风格：
- 默认：`color:#8e8e93 background:#f2f2f7`
- meeting：`color:#5856d6 background:rgba(88,86,214,0.10)`
- reading：`color:#ff9500 background:rgba(255,149,0,0.10)`
- journal 类不强制显示标签（保持简洁）

### 处理中顶部条

仅在有素材正在被 AI 处理时显示，处理完全部消失：

```
整理中  [🎙 录音 14:32 ⟳]  [📄 会议.docx ⟳]
```

- 背景 `#fafafa`，底部 `1px #f0f0f0`
- 每个素材一个小 chip，带 spinner

### 详情面板（右侧滑入）

点击日志条目后：
- Tauri 窗口向右展宽，右侧面板滑入（沿用现有 DetailSheet 机制）
- 面板内容：
  - 标题 + 日期/标签
  - 完整 markdown 内容（可编辑）
  - **原始素材** 区块：列出关联素材，支持播放（录音）/ 打开（文件）
  - **添加素材到此条目** 按钮

### 素材输入方式

1. **录音**：红色 FAB 按钮（保留现有交互）
2. **拖放文件**：拖到窗口任意位置触发，显示全屏蓝色遮罩提示
   - 支持：录音文件 / txt / md / pdf / docx
3. **（可选）菜单导入**

### 选中状态

选中条目：`background:rgba(0,0,0,0.055)` + `border-left:2px solid #ff3b30` + `padding-left` 缩减 2px

## 设置界面

新增配置项：

- **Workspace 路径**：日志和素材的存储根目录

- 录音功能（cpal + hound + afconvert 流程）
- 语音转写（DashScope STT，30秒以上自动触发）
- 深色模式支持
- 右键菜单（播放 / Finder / 删除）
- 最小窗口尺寸 280×400
