# 谨迹营销页改版设计文档

**日期：** 2026-03-30
**文件：** `daynote-design-visual.html`

---

## 背景

原页面使用单一 slogan「每一次思考，都值得被谨迹」贯穿全文，三个场景章节（个人日常、小组作业、需求评审）视觉格式混杂，缺乏清晰的受众对应关系。

改版目标：用**场景驱动共鸣**替代功能驱动说明——每个场景都有专属 slogan，让不同受众「一眼认出自己」。

---

## 整体结构

```
Hero
├── 场景一：个人思考
├── 场景二：学习
├── 场景三：办公会议
├── UI 截图
└── 信念板块
```

去掉原有的「工作方式变了」flip 过渡段和人类 vs AI 权限对比表。

---

## Hero 区

- **大标题：** 「每一次思考，都值得被谨迹」（通用 slogan，作为全页总纲）
- **副标题：** 录音、文件、文本丢进来 / 日志、待办、方案自动生成
- **前置痛点：** 保留现有的三行删除线痛点文案

---

## 场景一：个人思考

**Slogan：** 每一次思考，都值得被谨迹

**叙事格式：** 时间线

结构：
1. 时间线展示一天发生的事（录音、PDF、便签……堆积）
2. 18:01 一行：全部拖进谨迹，下班
3. 虚线分割
4. 次日打开谨迹——结果揭晓（结构化日志、待办清单、方案初稿）

---

## 场景二：学习

**Slogan：** 每一次嘱咐，都值得被谨迹

**叙事格式：** Before / After 对比

两列对比：
- **以前：** 课上拼命手写 / 还是漏了关键点 / 导师说的事记一半 / 截止日期全靠记忆
- **现在：** 打开录音专心听讲 / 课堂笔记自动生成 / 导师嘱咐变成待办 / 截止日期自动提炼

覆盖两个子场景：
1. **课堂录音** — 自动整理笔记、标注重点和考点
2. **导师嘱咐** — 面谈录音转化为待办清单 + 截止日期 + 行动提纲

---

## 场景三：办公会议

**Slogan：** 每一场会议，都值得被谨迹

**叙事格式：** Story Card 步骤流程

包含两个子故事：

### 子故事一：个人效率

一个人的一天：多场会议录音 → 拖进谨迹 → 次日结构化纪要 + 待办清单全部就绪。

### 子故事二：团队协作

Story Card 步骤（人类步骤用金色圆圈，AI 步骤用绿色实心圆）：
1. 开会讨论，打开录音
2. 把私域资料拖进谨迹
3. AI：会议纪要自动生成 + 公域资料自动收集
4. AI：按讨论框架编写文稿和 PPT
5. 二次会议，再次录音，讨论优化意见
6. AI：根据讨论自动优化每一处细节
7. ✓ 完成。没有人熬夜。没有人拼稿。

## 信念板块

三段引用式信念宣言，每段一句核心 + 一句展开：

| # | 核心句 | 展开句 |
|---|--------|--------|
| 1 | **它记住你说过的每一句话。** | 跨会议、跨日期，从历史素材中主动产出你需要的东西。 |
| 2 | **你的灵感，都应当得到重视。** | 任何零散的念头，不只是会议录音——每一个当下的想法都值得被捕捉和处理。 |
| 3 | **你的时间应该花在决策上，不是整理上。** | 整理录音、提炼待办、起草方案——这些不该是你的活。 |

## 视觉规范

与原页面保持一致：
- 深色模式为主，支持 light/dark 切换
- accent 色 `#c8933a`，green `#34d399`，red `#f87171`
- 字体：系统字体 + Noto Serif SC（标题）
- 无阴影、无渐变、内容主导
