## 背景

当前 README 严重过时——描述的是早期 v1「简单录音应用」阶段（nnnoiseless 降噪、音频处理流水线），而产品已演进为完整的 AI 知识管理工具「谨迹」。品牌定位、核心功能、技术架构均已发生重大变化。

---

## 设计决策

| 维度 | 决策 |
|---|---|
| 读者 | 两者兼顾：前半段面向用户，后半段面向开发者 |
| 语言 | 中文 |
| 结构 | 故事线结构（痛点 → 功能 → 上手 → 技术）|
| 视觉 | 静态截图一张，放在功能列表前 |
| 长度 | 精简型，约 100 行 |

---

## 内容结构

```
# 谨迹

## 为什么需要它

结论句

[截图]

3 步安装使用

---（分隔线，切换至开发者视角）

表格

```

---

## 关键写法原则

1. **slogan 直接点明产品主张**：「你负责思考，AI 负责剩下的」
2. **场景先行**：用三个真实痛点建立共鸣，再介绍功能
3. **HR 分隔线**（`---`）作为用户/开发者视角的自然切换，不加额外标题
4. **技术栈如实**：保留 nnnoiseless 等真实依赖，反映现状
5. **架构风格**与 CLAUDE.md 保持一致（ASCII 图）
6. **Releases 链接**：`https://github.com/quan2005/journal/releases`
