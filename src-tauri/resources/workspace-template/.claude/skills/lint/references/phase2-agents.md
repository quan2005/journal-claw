# Phase 2 并行 Agent Prompts

派出 5 个并行 Explore Agent，每个负责一个维度。下面是每个 Agent 的完整 prompt — dispatch 时整段作为 agent 任务，不要改动检查条目的顺序或措辞（这些是"什么算问题"的定义）。

**交叉验证契约**：Phase 1 缺陷清单必须作为输入传递给对应 Agent，Agent 必须首先验证清单中的每一项。如果 Agent 的结果与 Phase 1 清单矛盾（例如 Phase 1 发现了缺陷但 Agent 报告无发现），以 Phase 1 的事实为准，Agent 需解释差异原因。

---

## Agent 1：决策演进

```
扫描范围内的日志条目 frontmatter（tags + summary）。
找出：
- 新条目的决策是否推翻了旧条目的结论
- 同一主题是否出现了更新版本
- 截止日期已过但未记录结果的待办
输出：变更清单，每条标注 before→after、涉及文件路径、置信度
```

---

## Agent 2：人物与产品档案质量

```
扫描范围内的日志条目，交叉对比 identity/ 目录。
必须按 /identity-profiling skill 定义的规范检查（加载该 skill 获取五维度 / 四维度详细定义）。

找出：
- 条目中出现但没有 Identity 档案的人物（仅限有实质性互动的）
- 条目中描述的人物角色与档案不一致
- 档案内部存在矛盾（如同时写了两个不同的部门）
- 孤立档案：identity/ 中存在、但在扫描范围内的任何条目中均未被提及的人物档案
- 缺失关系交叉引用：两个人物在同一条目中共同出现，但两人档案中均未互相提及
- 【结构规范】人物档案是否按五维度组织，而非按会议日期追加段落
- 【结构规范】产品档案是否按四维度组织，「当前态势」是否标注了更新日期
- 【深挖不足】出现 3 次以上且有决策行为的人物，档案是否仍停留在初始化阶段（只有身份信息，缺少决策模式归纳）
- 【模式沉淀】扫描范围内，某个已建档人物反复出现的决策 / 表达 / 立场模式，满足三重验证（跨场景复现 ≥3 次 + 具有预测力 + 非角色通用行为）但档案中尚未写入对应归纳。输出时必须列出 3 条以上独立事件作为证据，并说明这条模式的"排他性"在哪里（为什么不是任何同类角色都会这样）。不满足三重验证的信号一律跳过，不输出低置信度猜测
- 【边界缺失】档案在「决策模式」「能力与盲区」「与用户的杠杆关系」维度已有归纳，但未写「观察边界」段落，或归纳明显超出证据范围（如从纯工作会议推断出私下态度）
- 【跨档案】产品级决策是否错误地展开在人物档案中（应归入产品档案的「战略判断与关键决策」）

输出：变更清单，每条标注档案路径、问题类型、建议修正、置信度
孤立档案只标注，不删除。缺失关系引用只在置信度高时（同一会议明确互动）才建议补充。
结构规范问题统一标注为中置信度。
```

## Agent 3：交叉引用

```
扫描范围内的日志条目的「关联记录」区。
找出：
- 同一天同主题的条目未互相链接
- 系列条目（标题含相同关键词如「墨宝落地」）未完整串联
- 条目正文引用了其他条目的内容但未建立链接
输出：缺失链接清单，每条标注源文件、目标文件、关联理由、置信度
```

---

## Agent 4：frontmatter 质量

```
扫描范围内的日志条目 frontmatter。

【先验证 Phase 1 缺陷清单，不可跳过】
{Phase 1 缺陷清单，包含文件路径和具体问题}

然后独立扫描，找出：
- summary 为空或无信息量（如「讨论了若干议题」）
- tags 不准确或缺失
- frontmatter 格式错误（如 `## tags:` 写成了 markdown 标题而非 YAML 字段、缺少 `---` 分隔符等）
- 文件不存在（Phase 1 检测到但实际已被删除/改名）

输出：修复清单，每条标注文件路径、当前值、建议值、置信度。
Phase 1 清单中的每一项必须出现在输出中（即使结论是"已修复"或"文件不存在"）。
```

---

## Agent 5：事实矛盾检测

```
扫描范围内的日志条目正文和 identity/ 档案。
找出：
- 同一事实在不同条目中有矛盾描述（如：A 条目说某产品定价 99 元，B 条目说 199 元）
- 同一人物在不同条目中有矛盾的角色 / 职位描述
- 同一决策在不同条目中有矛盾的结论（区别于「演进」—— 演进是有时间顺序的更新，矛盾是同时期的不一致）
输出：矛盾清单，每条标注文件 A、文件 B、矛盾内容、置信度。
无法确定是矛盾还是演进时一律低置信度跳过。
```
