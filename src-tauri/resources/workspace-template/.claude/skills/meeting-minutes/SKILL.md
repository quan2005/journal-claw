# 会议纪要整理

## 契约

| 维度 | 声明 |
|---|---|
| **reads** | `yyMM/raw/*`（原始素材）、`identity/*.md`（人物识别） |
| **writes** | `yyMM/*.md`（日志条目）、`identity/*.md`（新建/更新档案） |
| **depends** | `/identity-profiling`（涉及档案操作时） |

把会议素材（录音转写、粘贴文本、飞书妙记等）整理为结构化、可回溯的秘书稿。核心能力是根据会议性质自动选择最合适的纪要模板，确保信息保留率和逻辑完整度。

## 会议类型识别

阅读素材后，根据以下信号判断会议类型，选对应模板（从 `references/templates/` 加载）：

| 类型 | 识别信号 | 模板文件 |
|---|---|---|
| **决策会 / 评审会** | 多方参与、存在分歧或争论、需要达成共识、涉及资源分配或权限变更 | `argumentation-chain.md` |
| **周会 / 进度同步会** | 多人轮流汇报、商机 / 项目逐条过、有待办跟进项 | `progress-tracking.md` |
| **分享会 / 讲座** | 单人或少数人主讲、知识输出为主、听众提问为辅 | `knowledge-distillation.md` |
| **对齐会 / 碰头会** | 2-4 人小范围讨论、需求对齐或方案碰撞、介于决策会和同步会之间 | `alignment.md` |

**混合型会议**：如果一场会议混合多种类型（如周会中突然出现激烈争论），按议题拆分，每个议题独立选模板。

**无法判断**：默认 `alignment.md` — 它的结构最通用。

## 执行流程

1. **读素材** — 理解会议内容
2. **识别类型** — 按上表决策
3. **读模板** — 从 `references/templates/{模板文件}` 读入对应模板
4. **填充内容** — 按模板结构重组素材
5. **应用通用规则**（见下）

## 通用规则（适用所有模板）

### 引用原则

- 只保留改变讨论走向或暴露真实立场的原话
- 每个议题最多 3 条引用
- 附和性发言（"对对对"、"嗯"、"我同意"）不保留
- 转写质量差的引用，用书面语重述核心意思，不要硬搬错误百出的原文

### 人物处理

- 首次出现时标注角色 / 职位
- 后续用姓名即可，不重复标注
- 遵循 `/identity-profiling` 的规则创建 / 更新人物档案

### 素材质量应对

录音转写常见问题与应对：

- **声纹未识别** — 根据发言内容、称呼、上下文推断说话人，无法确认时标注 "⚠️ 说话人待确认"
- **粤普混杂 / 语音识别错误** — 理解意思后用书面语重述，常见错误映射如"新筹"→薪酬、"迹效"→绩效、"兜包"→豆包
- **断句错误** — 转写可能把一个人的连续发言切成多段，或把两个人的合并，按语义重新断句

### 灰色地带处理

"讨论了但没结论"的议题不能丢弃。放在「待确认」或「未对齐事项」中，明确标注各方的不同看法和下一步。
