# 会议纪要整理

将会议素材（录音转写、粘贴文本、飞书妙记等）整理为结构化、可回溯的秘书稿。核心能力是根据会议性质自动选择最合适的纪要模板，确保信息保留率和逻辑完整度。

## 会议类型识别

阅读素材后，根据以下信号判断会议类型，选择对应模板：

| 类型 | 识别信号 | 模板 |
|------|---------|------|
| **决策会 / 评审会** | 多方参与、存在分歧或争论、需要达成共识、涉及资源分配或权限变更 | 论证链模板 |
| **周会 / 进度同步会** | 多人轮流汇报、商机/项目逐条过、有待办跟进项 | 进度追踪模板 |
| **分享会 / 讲座** | 单人或少数人主讲、知识输出为主、听众提问为辅 | 知识提炼模板 |
| **对齐会 / 碰头会** | 2-4人小范围讨论、需求对齐或方案碰撞、介于决策会和同步会之间 | 对齐记录模板 |

如果一场会议混合了多种类型（比如周会中突然出现了激烈争论），按议题拆分，每个议题独立选择模板。

如果无法判断类型，默认使用对齐记录模板——它的结构最通用。

## 模板一：论证链模板（决策会 / 评审会）

适用于存在分歧、需要达成共识的会议。核心价值是保留决策的可回溯性——不仅记录结论，还记录结论是怎么来的。

### 每个议题的结构

```
### 议题标题

#### 争议焦点
一句话点明核心矛盾，让读者立刻进入语境。

#### 各方立场
列出关键人物的初始立场和核心关切，用标签标注立场倾向：
- **张三**（开放派）：主张...
- **李四**（收紧派）：担心...
- **王五**（折中派）：建议...

#### 关键论据
保留改变讨论走向的原话引用（每个议题不超过3条）。
只保留「改变了讨论走向」或「暴露了真实立场」的原话，
不保留附和性发言。每条引用附带上下文说明为什么这句话重要。

> "原话引用"

这句话的重要性在于...

#### 破局点
记录是什么打破了僵局——谁提出了折中方案、谁做出了让步、
什么案例说服了谁。

#### 共识
结论 + 负责人 + 时间节点。

#### 共识稳固度
标注这个共识的可靠程度：
- **坚实** — 全员认同，无保留意见
- **脆弱** — 有人让步但非完全认同，后续可能翻盘
- **易变** — 有方向无方案，需要持续对齐
```

### 论证链触发条件

并非所有议题都需要完整论证链。只有满足以下任一条件时展开：
1. 存在明确分歧（两人以上持不同立场）
2. 结论涉及后续资源分配或权限变更
3. 有人做出了让步或立场转变

无争议的信息同步类议题，即使出现在决策会中，也用简洁的结论式记录。

### 隐含信息标注

用 ⚠️ 标注"纪要不会写但你需要知道"的政治动态：
- 谁在帮谁解围
- 谁的让步是被施压的
- 某个结论对后续推进的潜在影响
- 情绪信号及应对建议

这些标注是纪要最有价值的部分之一——它们帮助读者预判后续推进的阻力点。

## 模板二：进度追踪模板（周会 / 进度同步会）

适用于多人轮流汇报、逐条过进展的会议。核心价值是快速提取状态变化和风险信号。

### 结构

```
## 一、进展汇总

按汇报人或项目线分节，每条用统一格式：

### 项目/商机名称
- **状态**：进行中 / 已完成 / 阻塞 / 关闭
- **本周进展**：一句话概括
- **风险/阻塞**：如有
- **下一步**：具体动作 + 负责人 + 时间

## 二、关键讨论

如果周会中出现了超出进度汇报的讨论（方向性决策、资源争夺等），
单独提出来，按论证链模板的简化版记录：
争议焦点 → 结论 → 负责人。

## 三、待办清单

从全文提取所有待办，汇总为一张表：

| 待办事项 | 负责人 | 截止时间 | 来源议题 |
|---------|--------|---------|---------|

## 四、风险看板

提取所有被提及的风险和阻塞项：

| 风险项 | 严重度 | 影响范围 | 当前应对 |
|-------|--------|---------|---------|
```

### 进度追踪的关键原则

- 状态变化比状态本身重要——"从60%推进到80%"比"目前80%"有信息量
- 数字要保留——金额、百分比、人数、日期，这些是周会纪要的骨架
- 商机类周会特别注意保留：商机阶段变化、客户关键反馈、竞争对手动态

## 模板三：知识提炼模板（分享会 / 讲座）

适用于单人主讲的知识输出场景。核心价值是从冗长的演讲中提炼可复用的认知资产。

### 结构

```
# 标题

> 时间、地点、演讲者信息

## 核心观点

用3-5个要点概括演讲者的核心主张，每个要点一句话。
这是整篇纪要的"电梯演讲"版本。

## 详细内容

按演讲的逻辑线索（而非时间线）组织：

### 主题一：标题
演讲者的论点 + 支撑案例 + 数据。
保留有冲击力的原话引用。

### 主题二：标题
...

## 案例索引

如果演讲中包含多个案例，单独列一张索引表：

| 案例 | 领域 | 核心启发 |
|------|------|---------|

## 个人启发 / 可迁移应用

从用户（许汝全）的视角，标注哪些观点可以迁移到当前工作中。
这一节用 💡 标记，与客观记录区分。
```

### 知识提炼的关键原则

- 按逻辑组织，不按时间组织——演讲者可能来回跳跃，纪要应该重新梳理逻辑线
- 案例是最有价值的部分——保留具体的公司名、产品名、数据指标
- 区分演讲者观点和客观事实——演讲者的判断用"演讲者认为"标注

## 模板四：对齐记录模板（对齐会 / 碰头会）

适用于小范围的需求对齐或方案碰撞。核心价值是记录"对齐了什么、没对齐什么"。

### 结构

```
## 背景

一句话说明这次对齐的目的和触发原因。

## 参会人

列出参会人及其角色/立场。

## 已对齐事项

按议题分节，每个议题：
- **结论**：一句话
- **关键依据**：为什么这么决定（如有争论，简要记录）
- **负责人**：谁来执行

## 未对齐事项

明确列出哪些问题没有达成共识：
- **分歧点**：各方的不同看法
- **下一步**：如何推进（再开会？找谁拍板？先试点验证？）

## 待办
```

### 对齐记录的关键原则

- "未对齐"比"已对齐"更重要——已对齐的事项执行就好，未对齐的才需要持续关注
- 记录每个结论的依据，哪怕只有一句话——避免后续有人说"当时没说过这个"

## 通用规则

以下规则适用于所有模板：

### 引用原则

- 只保留改变讨论走向或暴露真实立场的原话
- 每个议题最多3条引用
- 附和性发言（"对对对"、"嗯"、"我同意"）不保留
- 转写质量差的引用，用书面语重述核心意思，不要硬搬错误百出的原文

### 人物处理

- 首次出现时标注角色/职位
- 后续用姓名即可，不重复标注
- 遵循 identity 系统的规则创建/更新人物档案

### 素材质量应对

录音转写经常存在以下问题，需要主动应对：
- **声纹未识别**：根据发言内容、称呼、上下文推断说话人，无法确认的标注"⚠️ 说话人待确认"
- **粤普混杂 / 语音识别错误**：理解意思后用书面语重述，常见错误映射如"新筹"→薪酬、"迹效"→绩效、"兜包"→豆包
- **断句错误**：转写可能把一个人的连续发言切成多段，或把两个人的发言合并，需要根据语义重新断句

### 灰色地带处理

会议中经常出现"讨论了但没结论"的议题。这些不能丢弃，应该放在「待确认」或「未对齐事项」中，明确标注各方的不同看法和下一步。
