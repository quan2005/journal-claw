# 论证链模板（决策会 / 评审会）

**适用场景**：存在分歧、需要达成共识的会议。

**核心价值**：保留决策的可回溯性 — 不仅记录结论，还记录结论是怎么来的。三个月后翻回来读，能看懂当时为什么这么决定。

## 每个议题的结构

```
### 议题标题

#### 争议焦点
一句话点明核心矛盾，让读者立刻进入语境。

#### 各方立场
列出关键人物的初始立场和核心关切，用标签标注立场倾向：
- **张三**（开放派）：主张...
- **李四**（收紧派）：担心...
- **王五**（折中派）：建议...

#### 关键论据
保留改变讨论走向的原话引用（每个议题不超过 3 条）。
只保留「改变了讨论走向」或「暴露了真实立场」的原话，
不保留附和性发言。每条引用附带上下文说明为什么这句话重要。

> "原话引用"

这句话的重要性在于...

#### 破局点
记录是什么打破了僵局 — 谁提出了折中方案、谁做出了让步、
什么案例说服了谁。

#### 共识
结论 + 负责人 + 时间节点。

#### 共识稳固度
标注这个共识的可靠程度：
- **坚实** — 全员认同，无保留意见
- **脆弱** — 有人让步但非完全认同，后续可能翻盘
- **易变** — 有方向无方案，需要持续对齐
```

## 论证链触发条件

并非所有议题都需要完整论证链。只有满足以下任一条件时展开：

1. 存在明确分歧（两人以上持不同立场）
2. 结论涉及后续资源分配或权限变更
3. 有人做出了让步或立场转变

无争议的信息同步类议题，即使出现在决策会中，也用简洁的结论式记录。

## 隐含信息标注

用 ⚠️ 标注"纪要不会写但你需要知道"的政治动态：

- 谁在帮谁解围
- 谁的让步是被施压的
- 某个结论对后续推进的潜在影响
- 情绪信号及应对建议

这些标注是纪要最有价值的部分之一 — 它们帮助读者预判后续推进的阻力点。
