# 知识提炼模板（分享会 / 讲座）

**适用场景**：单人主讲的知识输出场景。

**核心价值**：从冗长的演讲中提炼可复用的认知资产。演讲会散，笔记要聚。

## 结构

```
# 标题

> 时间、地点、演讲者信息

## 核心观点

用 3-5 个要点概括演讲者的核心主张，每个要点一句话。
这是整篇纪要的"电梯演讲"版本。

## 详细内容

按演讲的逻辑线索（而非时间线）组织：

### 主题一：标题
演讲者的论点 + 支撑案例 + 数据。
保留有冲击力的原话引用。

### 主题二：标题
...

## 案例索引

如果演讲中包含多个案例，单独列一张索引表：

| 案例 | 领域 | 核心启发 |
|------|------|---------|

## 个人启发 / 可迁移应用

从用户的视角，标注哪些观点可以迁移到当前工作中。
这一节用 💡 标记，与客观记录区分。
```

## 知识提炼的关键原则

- **按逻辑组织，不按时间组织** — 演讲者可能来回跳跃，纪要应该重新梳理逻辑线
- **案例是最有价值的部分** — 保留具体的公司名、产品名、数据指标
- **区分演讲者观点和客观事实** — 演讲者的判断用"演讲者认为"标注
