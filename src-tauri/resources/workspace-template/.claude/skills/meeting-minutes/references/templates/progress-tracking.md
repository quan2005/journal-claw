# 进度追踪模板（周会 / 进度同步会）

**适用场景**：多人轮流汇报、逐条过进展的会议。

**核心价值**：快速提取状态变化和风险信号。读者关心的不是"此刻的状态"，而是"从上次到现在发生了什么"。

## 结构

```
## 一、进展汇总

按汇报人或项目线分节，每条用统一格式：

### 项目 / 商机名称
- **状态**：进行中 / 已完成 / 阻塞 / 关闭
- **本周进展**：一句话概括
- **风险 / 阻塞**：如有
- **下一步**：具体动作 + 负责人 + 时间

## 二、关键讨论

如果周会中出现了超出进度汇报的讨论（方向性决策、资源争夺等），
单独提出来，按论证链模板的简化版记录：
争议焦点 → 结论 → 负责人。

## 三、待办清单

从全文提取所有待办，汇总为一张表：

| 待办事项 | 负责人 | 截止时间 | 来源议题 |
|---------|--------|---------|---------|

## 四、风险看板

提取所有被提及的风险和阻塞项：

| 风险项 | 严重度 | 影响范围 | 当前应对 |
|-------|--------|---------|---------|
```

## 进度追踪的关键原则

- **状态变化比状态本身重要** — "从 60% 推进到 80%" 比 "目前 80%" 有信息量
- **数字要保留** — 金额、百分比、人数、日期，这些是周会纪要的骨架
- **商机类周会特别注意保留**：商机阶段变化、客户关键反馈、竞争对手动态
