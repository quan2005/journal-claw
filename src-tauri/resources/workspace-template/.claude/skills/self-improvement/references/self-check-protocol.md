# 素材处理后自检协议

处理完一份素材后，快速过以下 5 项检查。不需要向用户展示过程，只在发现问题时行动。

## 检查清单

### 1. Frontmatter 合规

- `tags` 是内联数组且首项为 `journal`？
- `summary` 非空、无引号包裹、1-2 句？
- `sources` 包含所有引用的原始素材路径？
- 开闭 `---` 完整？

**常见错误**：`## tags:` 而非 `tags:`、summary 用引号包裹、sources 遗漏素材。

### 2. 人物识别完整

- 素材中有实质性发言的人物是否都已识别？
- 已识别人物是否在 `identity/` 有对应档案（或判断为不需建档）？
- 新建档案是否用了 `.claude/scripts/identity-create`？
- 录音转写中的 speaker_id 是否正确绑定？

**常见错误**：遗漏次要但有实质发言的参会人、为只说附和词的 speaker_id 建空档。

### 3. 会议类型判断

仅对会议类素材检查：

- 选择的模板是否匹配会议实际类型？
- 分享会 vs 决策会的区分是否正确？（提问互动 ≠ 分歧讨论）
- 混合型会议是否按议题拆分了模板？

**常见错误**：把有少量提问的分享会误判为决策会。

### 4. Summary 信息量

- summary 是否结论先行？
- 是否有实际信息量？（"讨论了若干议题"是空话）
- 是否控制在 1-2 句？

**常见错误**：summary 过长、只描述过程不给结论。

### 5. 关联记录完整

- 同一天同主题的条目是否互链？
- 素材中提到的已有条目是否在关联记录区引用？
- 系列条目是否串联？

**常见错误**：遗漏同天相关条目的交叉引用。

## 自检结果处理

| 结果 | 动作 |
|---|---|
| 发现格式错误（frontmatter） | 立即修复 |
| 发现遗漏（人物、关联） | 立即补充 |
| 发现判断错误（会议类型） | 如果确信判断错误，修复并记录学习条目 |
| 发现质量问题（summary） | 立即改善 |
| 一切正常 | 不做任何记录，继续下一项工作 |

## 何时跳过自检

- 用户明确要求"快速处理"或"不用检查"
- 素材极短（一两句话的文本粘贴）
- 追加到已有条目（只追加了少量内容）
