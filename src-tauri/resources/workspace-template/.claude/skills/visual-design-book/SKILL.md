# Visual Design Book — 日志可视化设计说明书

## 这个 skill 做什么

读取一篇谨迹日志条目（`.md` 文件），将其内容转化为一个**单文件自包含的 HTML 页面**——一份用「逻辑推演」叙事方式呈现的可视化设计说明书。

**输入**：日志的 workspace 相对路径，如 `2604/12-产品评审会议.md`
**输出**：同目录下的 HTML 文件，如 `2604/12-产品评审会议-design-book.html`，并将该 HTML 路径写回日志 frontmatter 的 `sources` 字段。

## 触发方式

1. 日志详情右键菜单 →「转化为可视化设计」
2. Dock 栏输入：`/visual-design-book @2604/12-xxx.md`
3. 对话中提到"可视化设计""design book"并 @ 引用日志

## 执行流程

### 步骤 1：读取日志内容

```
workspace_path + 日志相对路径 → 读取完整 markdown 内容
```

解析 frontmatter（summary、tags）和正文。这是生成设计说明书的素材。

### 步骤 2：提炼叙事主线

从日志内容中找出：
- **北极星**：这篇日志的核心主题/结论是什么？
- **推演链**：内容中哪些概念/观点需要逐步展开？它们之间的因果关系是什么？
- **界面示意**：每个概念对应什么样的可视化呈现？

### 步骤 3：规划章节结构

列出章节标题（问句形式）和每章的左右内容概要。确保：
- 章节之间有递进关系，不是并列
- 每章回答一个问题，下一章在上一章的基础上追问
- 终章能把所有线索汇聚

### 步骤 4：生成 HTML

用纯 HTML + 内联 CSS 编写。参考 `references/structure-template.html` 获取完整模板。

**输出路径**：与源日志同目录，文件名为 `{DD}-{title}-design-book.html`

### 步骤 5：关联回日志

将生成的 HTML 文件的 workspace 相对路径添加到源日志 frontmatter 的 `sources` 字段：

```yaml
---
summary: 原有摘要
tags: [原有标签]
sources:
  - 2604/12-产品评审会议-design-book.html
---
```

如果 `sources` 字段已存在，追加而非覆盖。

## 叙事结构（五层）

### 第一层：Hero（定锚）
- **标题**：`{日志标题} · 设计说明书`
- **Slogan**：从日志 summary 提炼一句有节奏感的话
- **副标题**：补充说明

### 第二层：北极星愿景（战略锚点）
从日志核心论点出发，构建愿景卡片：核心理念 + 战略产出 + 终极目标。

### 第三层：逐章推演（核心）
每章是一个「问题 → 解答」推演单元：

```
┌─────────────────────────────────────────────────┐
│  ① 编号 + 标题（问句形式）                         │
│  ② 副标题（一句话回答）                             │
│ ┌──────────────────┐  ┌──────────────────┐      │
│ │  左：推理逻辑      │  │  右：产品示意      │      │
│ │  • 核心概念阐述    │  │  • UI mockup     │      │
│ │  • 机制说明        │  │  • 流程图         │      │
│ │  • 举例论证        │  │  • 数据看板       │      │
│ └──────────────────┘  └──────────────────┘      │
└─────────────────────────────────────────────────┘
```

**左侧写作模式**：概念标题 → 阐释段落 → 高亮引用框 → 细节要点 → 举例框
**右侧呈现模式**：纯 HTML/CSS 绘制 mockup（Tab栏、列表、对话流、流程图、矩阵、看板）

### 第四层：度量/治理章节组（可选）

### 第五层：终章收束
汇聚所有推演路径，回扣 Hero slogan。

## 视觉设计系统

### 色彩体系

使用 CSS 变量，支持明暗双主题。默认配色是温暖文雅的金橙色系：

```
主强调色: #C8933B（金橙）
文字层级: #3D352C → #635850 → #8A8078
背景层级: #FAFAF8 → #F5F4F1 → #EFEEE9
边框: #E5E3DD
```

色系应根据日志内容调性自适应——技术类用冷色、创意类用暖色、商业类用金橙。

### 排版规范

- 字体：`Noto Sans SC` + 系统 sans-serif
- 最大宽度：1100px 居中
- 章节间距：64px
- 卡片圆角：8px

### 必备交互

1. **明暗主题切换**：右上角按钮，`data-theme` 属性切换
2. **响应式**：768px 以下左右栏变上下
3. **打印适配**：`@media print` 隐藏切换按钮、每章分页

## 暗色主题技术要点

采用**快照恢复 + 两阶段占位符替换**机制：

1. `saveSnapshot()` 保存所有 inline style 浅色原始值
2. `applyDark()` 两阶段替换：源颜色 → `@@DARK0@@` 占位符 → 深色值（防止链式碰撞）
3. `restoreSnapshot()` 直接恢复快照
4. **绝对不要用双向 colorMap**

## 叙事质量检查

- [ ] 推演链是否连贯？
- [ ] 每章标题是否是问句？副标题是否回答了问题？
- [ ] 左右对照是否匹配？
- [ ] 终章是否回扣 Hero？
- [ ] 明暗主题反复切换 3 次无色彩残留？
- [ ] HTML 文件路径已写回日志 sources？

## 输出要求

- **格式**：单个 `.html` 文件，自包含
- **语言**：跟随日志语言
- **文件名**：`{DD}-{title}-design-book.html`（与源日志同目录）
- **关联**：HTML 路径写回源日志 frontmatter `sources`
